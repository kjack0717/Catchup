# catchup-ui-spec-v1 — UI·화면 설계 명세

> **위치**: Stage 5 화면 설계 기준선. `catchup-core-spec-v1`(기능 계약)의 시각·구현 짝.
> **플랫폼**: Expo (React Native). 아이보리+주홍 토큰, 홈=대시보드, 하단 4탭(스와이프+터치).
> **벤치마크**: 인스타(깔끔·부드러움) · 금천고 커뮤니티(대시보드·급식 레이아웃) · DC/아카라이브(커뮤니티 활성).

---

## §1. 디자인 토큰 (확정)

### 색상
| 토큰 | 값 | 용도 |
|------|-----|------|
| `bg` | `#FFFFF0` | 앱 배경(아이보리, RGB 255/255/240) |
| `surface` | `#FFFFFF` | 카드 표면 |
| `accent` | `#F57E4E` | 버튼·강조·활성 탭(주홍, RGB 245/126/78) |
| `accentDeep` | `#E4652F` | 눌림·강조 텍스트 |
| `admin` | `#D9A441` | 관리자 채팅 전용 골드 |
| `ink` | `#2A2622` | 본문 텍스트 |
| `inkSub` | `#6E655C` | 보조 텍스트 |
| `line` | `#EFE7D6` | 구분선·카드 테두리 |
| `danger` | `#D64545` | 신고·경고·D-day 임박 |

원칙: **배경은 아이보리, 텍스트/구조는 무채색, 색은 주홍에만 쓴다.** 골드는 관리자 채팅에서만 등장(희소성=식별성).

### 타이포 스케일
시스템 폰트 기반(가독·중립). 강조는 두께로 준다.
| 역할 | 크기/두께 |
|------|-----------|
| display (인사·D-day 숫자) | 28 / 800 |
| title (카드 제목) | 17 / 700 |
| body | 15 / 400 |
| caption (라벨·메타) | 12 / 500, `inkSub` |

### 간격·모양
- spacing 스케일: 4 · 8 · 12 · 16 · 24
- 카드: radius 16, 표면 `surface`, 테두리 `line` 1px, 그림자 약하게(인스타식 부드러움)
- 버튼: radius 12, 배경 `accent`, 텍스트 화이트, 눌림 `accentDeep`
- 탭바: 하단 고정, 활성 아이콘/라벨 `accent`, 비활성 `inkSub`

## §2. 시그니처 — "짜면 나오는 케첩"

브랜드 이중의미(catch up × ketchup)를 **한 곳에만** 쓴다.
- **당겨서 새로고침(pull-to-refresh)** = 케첩을 "짜는" 제스처. 당길 때 주홍 액센트가 병 입구에서 떨어지는 한 방울(dollop) 모션.
- 그 외 화면은 조용하게(무채색+주홍 최소). 시그니처가 유일한 볼드 포인트가 되도록 나머지는 절제.
- 빈 상태 카피도 브랜드 톤: "아직 짜낸 소식이 없어요. 당겨서 새로고침 해보세요."

## §3. 정보구조 & 내비게이션

하단 4탭 = 상단 위치의 material-top-tabs를 **하단 배치**해 스와이프+터치 동시 지원.

```
┌─────────────────────────────┐
│         (화면 콘텐츠)         │  ← 좌우 스와이프로 탭 전환
│                             │
├─────────────────────────────┤
│  🏫 나의학교  🎓 입시  💬 커뮤니티(●)  👤 프로필 │  ← 터치도 가능
└─────────────────────────────┘
```
- 커뮤니티 탭 라벨 옆 알림 뱃지(●숫자) = 새 쪽지. **커뮤니티 탭 한정.**
- 활성 탭 = 주홍, 비활성 = `inkSub`.

## §4. 홈 대시보드(나의 학교) 와이어프레임

```
┌─────────────────────────────┐
│  금천고 · 3학년              │  ← GreetingHeader
│  수능 D-131                  │     (display 숫자, 주홍)
├─────────────────────────────┤
│  📅 오늘 시간표               │  ← TimetableCard
│  1 국어 · 2 수학 · 3 영어 …   │
├─────────────────────────────┤
│  [ 배너: 공지/이벤트 ]        │  ← BannerCard (없으면 숨김)
├─────────────────────────────┤
│  🍚 오늘 급식                 │  ← MealCard
│  [중식] 흑미밥·김치찌개…       │     조/중/석 탭, 알레르기 정제
│  약 812 kcal                 │
├─────────────────────────────┤
│  🗓 학사일정                  │  ← ScheduleDdayCard
│  중간고사 D-12               │
├─────────────────────────────┤
│  🔥 우리 학교 인기글           │  ← HotPostsCard (상위 3)
│  1. … (♡24 💬8)             │
└─────────────────────────────┘
       ↕ pull-to-refresh = 케첩 짜기
```

각 카드는 **독립 로딩·실패 격리**: 급식 API가 죽어도 학사일정 카드는 뜬다. 빈/실패 상태는 카드 안에서 브랜드 톤 카피로 처리.

## §5. 컴포넌트 인벤토리 (8종)

| # | 컴포넌트 | 소속 | 상태 |
|---|----------|------|------|
| C1 | `GreetingHeader` | 홈 | 학교·학년·수능 D-day |
| C2 | `TimetableCard` | 홈 | NEIS 시간표(오늘) |
| C3 | `BannerCard` | 홈 | 공지/이벤트, 없으면 null |
| C4 | `MealCard` | 홈 | 급식(조/중/석·알레르기·열량) — 기존 `TodayCard` 계승 |
| C5 | `ScheduleDdayCard` | 홈 | 학사일정 D-day |
| C6 | `HotPostsCard` | 홈 | 인기글 상위 3(core-spec §2.3) |
| C7 | `PostCard` | 커뮤니티 | 태그·실명/익명·♡·💬 (T3) |
| C8 | `TagFilterBar` | 커뮤니티 | 태그 칩 필터 (T3) |

이번 트랜치(T2)는 C1–C6 + 탭 셸. C7–C8은 T3(커뮤니티 탭).

## §6. 화면별 상태 규칙
- **로딩**: 카드 높이 유지한 스켈레톤(레이아웃 점프 방지).
- **빈 상태**: 브랜드 카피 + 행동 유도("당겨서 새로고침").
- **실패**: 사과체 금지. "급식 정보를 불러오지 못했어요. 잠시 후 다시 시도돼요." (인터페이스 목소리)
- **접근성**: 최소 터치 44pt, 명도 대비 확보, `reduce motion` 존중(케첩 모션 비활성).
