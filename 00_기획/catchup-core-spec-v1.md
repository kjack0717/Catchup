# catchup-core-spec-v1 — 코어 기능·커뮤니티 상세 명세

> **위치**: Stage 4-A 하위 명세. `catchup-ui-spec-v1`(화면·컴포넌트)의 상위 기능 계약서.
> **원칙**: 초기엔 비(非)커뮤니티 기능(급식·D-day·시간표·계산기)으로 리텐션 확보 → 점진적 커뮤니티 활성화.
> **출처**: 1차 기획회의(2026-07-10) + 마스터 파이프라인 v3.

---

## §0. 정보구조(IA) — 확정

하단 4탭. **스와이프 + 탭 터치 병행** 이동.

| # | 탭 | 성격 | 핵심 내용 |
|---|-----|------|-----------|
| ① | **나의 학교(홈)** | 대시보드 | 인사·시간표·배너·급식·학사일정(D-day)·인기글 카드 집약 |
| ② | **입시 정보** | 도구 | 내신 계산기 · 모의고사 컷/등급 · 모집요강 링크 |
| ③ | **커뮤니티** | 소통 | 태그 단일 피드 · 실명/익명 · 쪽지 뱃지 |
| ④ | **내 프로필** | 개인 | 프로필·내 글·설정·문의·탈퇴 |

- 쪽지(DM) 알림 뱃지는 **커뮤니티 탭에만** 표시(전역 아님).
- "홈 = 대시보드"는 나의 학교 탭이 여러 카드를 얹는 상위 컨테이너라는 뜻.

## §1. 홈 대시보드 (나의 학교) — ★1

카드 스택(위→아래). 각 카드는 독립 로딩·실패 격리.

1. **인사 헤더**: "○○고 3학년 · 오늘 D-□□(수능)" 형태. 졸업생이면 학사모 배지.
2. **오늘 시간표**: NEIS `hisTimetable` 기반 오늘 교시. (사용자 커스터마이징은 §4 참조)
3. **배너**: 공지/이벤트(운영자 편집). 없으면 숨김.
4. **급식 카드**: NEIS `mealServiceDietInfo`. 조·중·석식 탭, 알레르기 번호 정제, 열량 표기. → `TodayCard.jsx` 재사용(구현 ✅).
5. **학사일정·D-day 카드**: NEIS `SchoolSchedule`. 시험·수능 D-day 자동 계산. → 구현 ✅.
   - ⚠️ **수능 확정일은 한국교육과정평가원 공고로 대조 후 확정**(neis-core.mjs의 추정값 교체).
6. **인기글 카드**: 우리 학교 실시간 HOT 상위 3(§3.4 로직).

## §2. 커뮤니티 정체성 — 확정

- **실명 기본 / 글쓰기 시 [익명] 토글.** 기본은 본명 노출, 작성 화면에서 익명 선택 가능.
- **게시판 분리 ❌ → 태그 단일 피드.** 하나의 피드에 글이 흐르고, 태그로 필터.
- 학교 격리: 같은 학교 인증회원만 서로의 글 열람·작성(RLS로 강제, Gate 3 통과).

### §2.1 태그 (초기 세트)
`잡담` · `입시정보` · `분실물` · `수행평가` · `학교`
- 글 작성 **전** 유형 태그 1개 선택(필수). 다중 태그는 v1.1 이후 검토.
- 피드 상단 태그 칩으로 필터. 기본은 전체.

### §2.2 익명 식별 규칙 — 확정 (에브리타임식)
게시글 **1건 단위**로 익명 순번을 부여한다(글 전역 아님).

- 글 작성자가 익명이면 본문 작성자는 **"글쓴이"**로 표기.
- 댓글·대댓글에서 익명 사용자는 **해당 글 안에서 처음 등장한 순서대로** `익명1`, `익명2`… 부여.
- 같은 사용자가 같은 글에서 다시 쓰면 **동일 순번 유지**(글 범위 내 안정적 매핑).
- 다른 글에서는 매핑이 초기화(글 간 연결 불가 → 추적 방지).
- 내부적으로는 `(post_id, user_id) → seq` 매핑을 저장하되 **화면엔 seq만** 노출.

### §2.3 인기글 / 실시간 HOT — 로직 확정
학교별로 계산. **롤링 24시간** 윈도우.

```
score = likes + 2 × comments        // 댓글 가중 2배(참여 신호)
window = 최근 24h 작성글
HOT 진입 기준: score ≥ 5 AND likes ≥ 3   // 초기 임계치, 학교 규모 따라 튜닝
정렬: score DESC, 최신 DESC
홈 카드: 상위 3 / 커뮤니티 HOT 탭: 상위 20
```
- 신규 학교(활동 적음)는 임계치 자동 완화(예: 인증회원 <30명이면 `score ≥ 2`).
- 삭제·블라인드 글은 즉시 제외. 캐시 갱신 주기 5분.

### §2.4 상호작용
- **좋아요** / **스크랩**(내 프로필 → 스크랩함에서 다시 보기).
- **댓글 + 대댓글**(1단계 depth). 대댓글은 부모 댓글에 종속.
- 정렬: 등록순 기본, 인기순(좋아요) 토글.

### §2.5 쪽지(DM)
- 1:1 경량 채팅. 커뮤니티 글/프로필에서 진입.
- 알림 뱃지는 커뮤니티 탭 하단에 숫자 표시.
- 대량 사용 가정 안 함(자료 공유 중심) — 단, 사용 패턴은 단정하지 않고 운영 중 관찰.

### §2.6 관리자 채팅 — 확정
- 쪽지 목록 **최상단 고정 + 골드 계열 UI**(다른 채팅과 시각적 구분).
- **일방향**: 공지·안내(운영자→회원) + 가벼운 피드백·문의(회원→운영자) 접수.
- 접수 문의는 **AI 자동 정리/분류** 후 운영자 처리(문의 게시판 Q&A는 미채택).
- 만료 예고·탈퇴 안내 등 시스템 공지도 이 채널 사용.

### §2.7 졸업생 — 확정
- 활동 기간: **졸업 후 2년**.
- 닉네임 옆 **학사모 배지**(실명 글에만 표시). 탭하면 "○회 졸업생".
- 만료 **사전 알림**(관리자 채팅) → 만료 후 **자동 탈퇴**.
- 3월 재인증 사이클과 연동(졸업생 정리, Stage 9).

### §2.8 탈퇴 정책
- 회원 탈퇴 시 **작성글 삭제 여부는 본인 선택**(기본값: 남김).
- 탈퇴 후 삭제 요청은 별도 문의(관리자 채팅/이메일)로 접수.

## §3. 입시 정보 탭 — ★3

1. **내신 등급 계산기(단순형)**: 평균 + 단위수 반영, **과목 선택** 지원.
   - **입력 편의 최우선**(회의 핵심 지적): 과목·성적 입력 UX를 최대한 간소화. 파일/생기부 업로드는 ❌(저작권·프라이버시).
2. **모의고사 컷·등급**: 표준점수·원점수·등급 표. 반자동(시즌마다 AI 프롬프트로 갱신, 월 1회 수준).
3. **모집요강 링크**: 대학 입학처 링크 목록.
   - 자동 팔로우·크롤링은 **기술적 비현실 → 기각.** 수동/AI 보조 갱신.
4. **입시 뉴스 큐레이션**: 🔁 보류(토큰 비용). 후순위 디벨롭.

## §6. 안전 방어선 (모든 기능에 동시 부착) — Stage 4-B

> ⚠️ **이 서비스의 이용자 다수가 고등학생(미성년)이다. 아래 보호장치는 운영자 연령·명의 상태와 무관하게 유지한다.** 커뮤니티 기능을 붙일 때마다 세트로 붙는다.

| # | 방어선 | 동작 |
|---|--------|------|

| 4-2 | 신고 자동 블라인드 | 동일 글 신고 3건 → 자동 블라인드 + 검토 큐 진입 |
| 4-3 | 차단·탈퇴 | 사용자 차단(스토어 심사 필수), 회원 탈퇴 |
| 4-4 | 피해자 동선 | 즉시 숨김 + 상담 연결(117 학교폭력 / 1388 청소년) |
| 4-5 | 자해 암시 대응 | 상담 안내(1388 / 109 자살예방) 노출, 관련 콘텐츠 확산 억제 |

## §7. 데이터 모델 요약 (구현 기준선)

기존 `catchup-schema-rls.sql`에 커뮤니티 상세 테이블 추가/확장:

| 테이블 | 핵심 컬럼 | 비고 |
|--------|-----------|------|
| `posts` | id, school_code, author_id, is_anonymous, tag, body, created_at, blinded | RLS: 같은 school_code만 |
| `post_anon_seq` | post_id, user_id, seq | (post_id,user_id) 유니크 → 익명 순번 §2.2 |
| `comments` | id, post_id, parent_id(null=댓글), author_id, is_anonymous, body | 대댓글 1-depth |
| `likes` | post_id, user_id | 유니크(중복 방지) |
| `scraps` | user_id, post_id | 개인 스크랩함 |
| `messages` | id, from_id, to_id, body, read_at, is_admin | 관리자 채팅 = is_admin |
| `reports` | id, target_type, target_id, reporter_id, reason | 3건 트리거 §6-4-2 |
| `hot_cache` | school_code, post_id, score, computed_at | 5분 캐시 §2.3 |

> 정확한 DDL은 `catchup-moderation-schema.sql`(신고/블라인드/제재)과 병합해 마이그레이션으로 관리.
